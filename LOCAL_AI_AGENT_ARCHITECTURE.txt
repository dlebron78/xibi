The AI agent does not consistently execute tools when interacting through Telegram.

Observed behavior:

• agent acknowledges commands
• agent suggests running commands
• agent does not emit actual tool calls

Example failure:

tool exists
allowlisted
verified working

Logs show:

16. Current Hypothesis

The issue may be caused by:

Model limitations

Small models (7B–8B) may struggle with:

• structured tool calls
• agent orchestration
• execution formatting

Skill configuration

It is also possible that OpenClaw skills were not properly enabled during onboarding.

During onboarding several options were skipped, including:

skills installation
dependency setup
hooks configuration

This may prevent proper tool routing.
