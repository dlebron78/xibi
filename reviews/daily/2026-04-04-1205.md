## Bug fix applied this run

**Root cause:** `tasks/triggered/` did not exist in the git repo. Git does not track empty
directories, so the directory was never created on NucBox clones. The trigger script uses:

```bash
TRIGGERED_COUNT=$(find "${XIBI_DIR}/tasks/triggered" -name "*.md" 2>/dev/null | wc -l)
```

With `set -euo pipefail`, the `2>/dev/null` suppresses find's stderr but the pipe still
exits with find's exit code (1) when the path doesn't exist. With `pipefail`, this kills
the entire script after `Auto-detected pending task:` — silently, because `set -e` doesn't
print a message.

**Fix:** Added `tasks/triggered/.gitkeep` to the repo (commit `93d4f80`). NucBox pulled the
fix; the trigger script now advances past the gate check.
