### Vision notes

1. **Backward-compat stub path in react.py line 112**: The fallback code `tool_manifest = next((t for t in skill_registry if t.get("name") == tool_name), None)` treats skill_registry as a flat list of tool manifests, but post-PR it is a list of skill-level manifests with nested 'tools'. This path is only reached when `executor=None` (legacy step-02 backward compat). In production, executor is always non-None. Not a regression — 774 tests pass — but worth cleaning up in a future step.

2. **MCPToolManifest naming**: Python attribute `input_schema` (snake_case) stores value used under `inputSchema` key in synthetic manifests. Intentional Python-vs-JSON naming split. Clear, correct.

3. **DEFAULT_TIER RED is global**: The change from GREEN to RED applies to any unknown tool, not just MCP tools. This is correct per architecture ("any first-time action type → Red tier") and is a safety improvement, but it is a broader change than MCP alone.

4. **No retry logic for MCP tool failures**: call_tool() doesn't retry on transient failures. Acceptable for this foundation step — circuit breaker handles persistent failures.
