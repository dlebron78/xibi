### Why step-37: Wire LLM Routing Classifier

The highest-value P1 backlog item after deployment is the semantic routing fallback.
`xibi/routing/llm_classifier.py` already contains a complete, tested `LLMRoutingClassifier`
(5 tests in `tests/test_classifier.py`). The implementation uses `get_model("text", "fast")`
with `generate_structured()` to map a query to a skill/tool when BM25 confidence is below
threshold. It handles hallucination (verifies skill+tool exist in manifests), low confidence
(returns None), and model errors (returns None silently).

**The gap:** the classifier is never called. `react.run()` has no `llm_routing_classifier`
param. `TelegramAdapter` doesn't hold a classifier instance. `__main__.py` doesn't instantiate
one. Queries that BM25 cannot match (paraphrased phrasings, novel intents) go to full ReAct
with no routing hint at all.

**The fix is pure wiring:** 3 files changed, no new classes, no logic redesign. This directly
improves routing quality for Daniel's daily Telegram queries now that Xibi is live on NucBox.

The second P1 backlog item (per-user Telegram session isolation) is already implemented —
`_get_session()` uses `telegram:{chat_id}:{date}` as the session key, and sessions are stored
per `chat_id` in `self._sessions`. What's missing is an integration test, which can be a
smaller follow-up or added by Jules as part of a future step.
