Since step-36 set workdir to `~/.xibi/`, this resolves to a nonexistent file. The tools also use bare `sqlite3.connect()` instead of `open_db()`, bypassing WAL mode and timeout settings. The memory system is write-only from the model's perspective.

### Step-42 spec rationale

This is a production correctness fix with high user-facing impact. Every explicit `remember` invocation silently errors. Every `recall` invocation either errors (bregger.db not found) or returns nothing from beliefs compressed by session compression. The model cannot use the memory system at all.

The spec includes:
1. Fix `bregger.db` → `xibi.db` in all five tools
2. Replace bare `sqlite3.connect()` with `open_db()` (WAL/timeout)
3. Update `recall.py` to search both `beliefs` (session-compressed) and `ledger` (explicit) tables
4. Update `archive.py` to expire beliefs by setting `valid_until`
5. Change `BREGGER_WORKDIR` fallback env var to `XIBI_WORKDIR`
6. 8 tests using real `xibi.db` via `init_workdir()` — no mocks

**No schema changes.** Both `beliefs` and `ledger` tables already exist in `xibi.db` since migration 1.
