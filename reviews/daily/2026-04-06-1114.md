## Step-56 Spec Written

**Title:** Wire nudge() Into Executor: Core Tool Auto-Registration

**Root cause diagnosed:** `Executor.execute()` resolves tools via `SkillRegistry.find_skill_for_tool()`.
The deployed `~/.xibi/skills/` directory contains email + search but NOT nudge. The nudge skill
exists in `xibi/skills/sample/nudge/` but the sample directory only serves as a full fallback
when the user's skills_dir doesn't exist — it is not merged. So `find_skill_for_tool("nudge")`
returns `None` → "Unknown tool: nudge".

**Fix:** Add `_register_core_skills()` to `Executor.__init__`. After user registry loads, scan
`xibi/skills/sample/` for skills not present in user registry and register them. User skills
always win (no overwrite). Never raises — wrapped in try/except. Applies to both `Executor` and
`LocalHandlerExecutor` since the subclass calls `super().__init__()`.
