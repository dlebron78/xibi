## Step-28 rationale
With condensation in place (step-27), the next architectural gap is the command layer.
Tool calls currently bypass schema validation, have no permission tiers, and can fire
duplicate nudges in rapid succession during observation cycles. Step-28 (roadmap Step 7)
adds three interlocking mechanisms: schema validation gate, Green/Yellow/Red permission
tiers, and nudge dedup. The implementation wraps the existing `Executor` without modifying
it — all new logic lives in `xibi/command_layer.py` + `xibi/tools.py`. The `react.py`
change is additive (new optional `command_layer=` param, defaults to None = no change).

This is the architectural prerequisite for Step 8 (Observation Cycle MVP) — the observation
cycle needs Red blocking (non-interactive) and nudge dedup to be safe to run autonomously.
