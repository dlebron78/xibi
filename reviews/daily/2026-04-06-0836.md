## Step-54 Spec: GitHub MCP Source (Read-Only Repository Activity)

**Rationale:** Step-53 proved the stdio subprocess lifecycle for a no-auth server.
GitHub is the next server in the MCP sequence: read-only, API-key auth (`GITHUB_TOKEN`
env var, no OAuth). This step validates the env-var-secret injection pattern that all
future token-authenticated MCP servers (Slack, Notion, Linear) will follow.

**Scope:**
- `github_activity` signal extractor in `extractors.py` (commits, issues, PRs → signals)
- `_sha_to_ref_id()` and `_issue_to_ref_id()` helpers for stable dedup
- `_poll_watch_repos()` in `source_poller.py` for profile-driven repository polling
- `watch_repos` profile schema (repo, watch_commits/issues/prs, interval_minutes, max_items)
- GITHUB_TOKEN validation at poll time with WARNING log on missing token
- 14+ tests in `tests/test_github_source.py`
- No new DB tables, no schema migration
