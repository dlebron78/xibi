## Step-43 Spec — Rationale

### Why this step next

The project memory (MEMORY.md) lists native function calling as the highest-leverage
next priority. The infrastructure already exists: `OllamaClient.generate_with_tools()`
calls Ollama's `/api/chat` endpoint with a structured tools list and returns parsed
`tool_calls` dicts. Currently `react.py` ignores this entirely, using text parsing
(JSON/XML/text) that requires parse recovery on failure and still misses novel phrasings.

Native function calling eliminates parse failures entirely for models that support it.
No JSON extraction regex. No recovery prompt retries. The model either returns a valid
tool call or it returns content — both cases are handled cleanly.

### What the spec builds

1. **`BreakerWrappedClient` extension** (`router.py`): `generate_with_tools()` proxy and
   `supports_tool_calling()` capability check — 15 lines, low risk.

2. **React loop native mode** (`react.py`): `_build_native_tools()` (flattens skill
   registry + adds `finish`/`ask_user` as real tool schemas), `_native_step()` (calls
   `generate_with_tools()` and maps response to tool_name/tool_input), message history
   management helpers, native branch in main loop.

3. **Auto-detection fallback**: If `react_format="native"` is configured but the model
   lacks `generate_with_tools`, silently downgrade to json and log a warning. No crashes.

4. **Config wiring**: Optional `react_format` key in `config.json`. Telegram adapter and
   `__main__.py` read it and pass to `react_run()`. Existing deployments without the key
   are unaffected (default: json).

5. **18 tests**, all mocked — no real LLM calls.

### What does NOT change
- `json`, `xml`, `text` formats fully preserved.
- `generate_with_tools` implementation untouched.
- No schema changes, no new dependencies.
- Gemini/OpenAI/Anthropic clients unaffected — they fall back gracefully.

### Key design decisions in the spec
- The spec is explicit that `_native_messages` (chat history for the model) and
  `scratchpad` (Step list for pipeline control) are parallel, not merged. The stuck/repeat
  detection operates on `scratchpad`, same as json mode.
- `finish` and `ask_user` are injected as first-class tool schemas in the tools list —
  the model calls them as tools instead of embedding JSON in freeform text.
- The assistant tool-call message must use Ollama's wire format for the tool_calls field
  when appending back to the message history.
