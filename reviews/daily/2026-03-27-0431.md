## CI Fix Details — PR #15

Three failure categories fixed:

**1. `sqlite3.OperationalError: unable to open database file` (17 tests)**
Root cause: `open_db()` in `xibi/db/__init__.py` calls `sqlite3.connect(db_path)` without
ensuring parent directory exists. In CI, `Path.home() / ".xibi" / "data"` doesn't exist.
Fix: Added `db_path.parent.mkdir(parents=True, exist_ok=True)` at the top of `open_db()`.
Affects: all executor tests, cli tests, router integration tests that exercise CircuitBreaker.

**2. `KeyError: 'message'` (2 tests)**
Root cause: `executor.execute()` changed error return key from `"message"` to `"error"`
for tool-not-found and circuit-open paths (using `XibiError.user_message()`), breaking
existing tests that check `result["message"]`.
Fix: Reverted those paths to use `"message"` key with a plain string, keeping `_xibi_error`
for internal consumers. The `_execute_inner` method already used `"message"` — making the
outer `execute()` consistent.

**3. `xibi.errors.XibiError` raised instead of `RuntimeError` (2 tests)**
Root cause: PR intentionally changes `OllamaClient` and `GeminiClient` to raise `XibiError`
instead of `RuntimeError` — an API change. Tests `test_client_timeout_handling` and
`test_gemini_client_generate_exception` still expected `RuntimeError`.
Fix: Updated both tests to `pytest.raises(XibiError)` and check `.message` attribute
directly (`"timed out"` and `"Gemini call failed"` respectively).

All 526 tests pass locally after the fix.
